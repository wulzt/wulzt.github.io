Vue

  • 生命周期
    create、mount、update、destory
  • create： 初始化事件，进行数据的观测，created时数据已经和data属性进行绑定
  • mount： 给vue实例对象添加$el 成员，替换掉挂载的DOM元素
  • update： data中的数据发生了改变，触发对应组件额度重新渲染
  • destory： 销毁

Vue 实例有一个完整的生命周期，也就是从开始创建、初始化数据、编译模版、挂载Dom -> 渲染、更新 -> 渲染、卸载等一系列过程，我们称这是Vue的生命周期。

  • 组件间的通信

  1. props/$emit
    父组件通过props向下传递数据给子组件。注： 组件中的数据共有三种形式：data、props、computed
    子组件通过事件形式向父组件发送消息

  2. eventBus
    通过一个空的vue实例作为事件中心，用它来触发和监听事件

    • 中间件bus.js 创建一个新的vue实例 export default new vue()
    • 要用到的组件里import
    • 组件1触发事件 Bus.$emit(‘getTarget’,event.target)
    • 组件2监听事件 Bus.$on(‘getTarget’,target=>{console.log(target)})

  3. vuex
    项目比较大时选择更好的状态管理解决方案vuex
    $store.dispatch(‘action名称’,data)来触发
    刷新之后会回到初始状态，要和localstorage一起使用

  4. $attrs/$listeners
    多级组件嵌套需要传递数据

  • $attrs 包含了父作用域不被prop所获取的特性绑定，可以通过v-bind=“$attrs”传入内部组件
  • $listeners 包含了父作用域中的v-on事件监听器

  1. provide/inject
    需一起使用，以允许一个祖先组件向其所有子孙后代注入一个依赖，不论组件层次有多深，并在上下游关系成立的时间里始终生效
    祖先组件： provide:{name:’tom’}
    子孙组件： inject: [‘name’]
    注意： 绑定并不是可响应的

  2. $parent.$children & ref

  • ref： 如果在普通DOM元素上使用，引用指向的就是DOM元素，如果在子组件上，引用就指向组件实例
  • $parent / $children: 访问父/子实例

ref访问组件

  • virtual dom
    是用js对象结构来表示dom树的结构，然后用这个树构建一个真正的dom树。当状态发生变更的时候重新构建一棵新的对象树，对比新树和旧树的差异，将所记录的差异应用到所构建的真正的dom树上。virtual dom本质是在js和dom之间做了一个缓存

优势：

  1. 虚拟dom进行频繁修改然后一次性比较并修改真实dom中需要修改的部分，最后在真实dom中进行重排重绘，减少过多dom节点重排重绘损耗
  2. 有效降低大面积真实dom的重排重绘，因为最终与真实dom比较差异，可以只渲染局部

  • 双向绑定原理
    利用Object.defineProperty劫持对象的访问器。给对象绑定getter、setter函数，属性变化的时候触发setter函数，获取变化，根据变化去做出后续响应

  • 响应式系统

  • 任何一个vue component都有一个与之对应的watcher实例
  • vue的data上的属性会被添加getter和setter属性
  • 当vue component render函数被执行时，data会被读，getter函数被调用，这时会把watcher添加到相应的dep上，是一个依赖收集的过程
  • data被改动时，setter方法会被调用，这时vue去通知所有依赖于此组件去调用它们的render函数进行更新

  • diff检测差异
    据响应式系统，通常绑定一个数据就需要一个watcher，一旦我们绑定细粒度过高就会产生大量的watcher，会带来内存和依赖追踪的开销，而细粒度过低又无法精准侦测变化，因此vue的设计时选择中等细粒度的方案，在组件级别进行侦测。通常我们会第一时间侦测到发生变化的组件，然后在组件内部进行virtual dom diff获取更加具体的差异

  • object.defineProperty和proxy

  • object.defineProperty缺陷：

  1. 无法监听数组的变化，以下八种方法可以，但vm.items[indexOfItem] = newValue这种是无法检测的

  2. 只能劫持对象的属性，如果属性值也是对象那么需要深度遍历

  • proxy特点
    ES6正式发布，它在目标对象之前架设一层“拦截”，外界对该对象的访问必须先通过这层拦截
  1. 可以监听对象
  2. 可以直接监听数组变化
  3. Proxy返回的是一个新对象,我们可以只操作新的对象达到目的,而Object.defineProperty只能遍历对象属性直接修改
