  • transition和animation区别
  1. 触发条件：transition需要由事件触发(hover)，animation不需要
  2. 循环： animation可以设定循环次数
  3. 精确性： animation可以设定每一帧的样式和事件，transition只能设定头尾
  4. 与js的交互： 用js灵活设定动画属性，用transition
  • 不定高的div居中
  1. position: absolute; top: 0; bottom: 0; margin: auto;
  2. position: absolute; top: 50%; transform: trnslateY(-50%);
  3. display: flex; align-items: center;
  • 浮动
    浮动元素会从普通文档流中脱离，但浮动元素影响的不仅是自己，它会影响周围元素对其进行环绕；不管一个元素是行内元素还是块级元素，如果被设置了浮动，那浮动元素会生成一个块级框，可以设置它的宽度和高度。
    清除浮动（解决父元素高度塌陷）：
  • 给父元素定义高度
  • 让父元素也浮动
  • 父元素display: table;
  • 父元素设置overflow: hidden;
  • clearfix：使用内容生成的方式清除浮动
  • css选择器分类

优先级：!important > 行内样式 > id > class > 标签 > 通配符* > 继承 > 浏览器默认属性

  • 为什么css放顶部而js写末尾

  • 浏览器预加载css后，可以不必等待html加载完毕就可以渲染页面了
  • js写在尾部，主要是因为js主要扮演事件处理的能力，一方面很多操作是在页面渲染后才执行；另一方面可以节省加载时间，使页面能更快加载，提高用户的良好体验

布局

  • flex弹性布局
    可以简单地使一个元素居中，栅格化系统布局
    设为flex布局后，子元素的float、clear和vertical-align属性将失效
    水平的主轴 main axis
    垂直的交叉轴 cross axis

  • 三栏布局

  • 左边定宽右边自适应
  • 左浮动，右边宽度设置100%。 .left{float: left} .right{width: 100%;}
  • 父容器设置display: flex; right部分设置flex: 1

  • 水平居中
    行内元素居中 父元素text-align: center;
    块状元素居中 宽度一定用margin: auto; 宽度不一定，块级变行内，父上用text-align

  • BFC
    块级格式化上下文，它是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素
    哪些元素会生成BFC：

  • 根元素
  • float属性不为none
  • position 为absolute或fixed
  • display为inline-block，table-cell，table-caption，flex，inline-flex
  • overflow不为visible
  • 浮动
    float
    浮动会造成父元素高度塌陷问题，一个块级元素如果没有设置height，其height由子元素撑开，对子元素使用了浮动后，子元素会脱离文档流，也就是说父元素没有内容可以撑开其高度，这样父元素的height会被忽略，这就是所谓高度塌陷

    伪元素： 用于创建一些不在文档树中的元素，并为其添加样式
